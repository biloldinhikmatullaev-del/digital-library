# Lumina — Клон интернет-магазина (React + Firebase + Netlify Serverless)

**Lumina** — это премиальный, современный прототип/клон интернет-магазина дизайнерской электроники и девайсов для рабочего места. Проект разработан с использованием **React (Vite)**, **Firebase** (для авторизации и базы данных) и бессерверных функций **Netlify (Node.js)** для бэкенд-логики.

Сайт оформлен в эстетичном стиле темной темы с использованием **Vanilla CSS** (переменные HSL, стеклянный эффект, адаптивные сетки и микроанимации).

---

## Архитектура и стек технологий

- **Фронтенд**: React (Vite)
- **Стилизация**: Vanilla CSS (переменные, адаптивная сетка, плавные эффекты наведения, анимация выезжающей корзины)
- **Бэкенд (Serverless)**: Netlify Functions (Node.js) для обработки API-запросов (`netlify/functions/`):
  - `products.js`: возвращает список товаров.
  - `checkout.js`: имитирует обработку оплаты (например, Stripe) и сохраняет заказ.
- **База данных и авторизация**: Firebase Client SDK (Authentication + Firestore).
- **Режим оффлайн-заглушек (Mock Mode)**: Если ключи Firebase не настроены, проект автоматически переключается в локальный режим работы, используя `localStorage` для хранения данных пользователей и истории заказов, а также локальный каталог товаров. Это позволяет сразу запустить и протестировать весь функционал.

## Быстрый старт

### 1. Установка зависимостей

Установите все необходимые npm-пакеты в корневой папке проекта:

```bash
npm install
```

### 2. Запуск в режиме разработки

Чтобы запустить фронтенд вместе с бессерверными функциями Netlify на одном локальном порту, используйте Netlify CLI:

```bash
# Автоматически загрузит Netlify CLI и запустит проект
npx netlify dev
```

После этого сайт откроется по адресу `http://localhost:8888`. Все запросы к `/.netlify/functions/*` будут корректно обрабатываться локальными Node.js-функциями.

*Вы также можете запустить отдельно фронтенд (без поддержки серверных API) с помощью команды:*
```bash
npm run dev
```

## Интеграция с Firebase

Для перевода приложения из демонстрационного режима (Mock Mode) в рабочий режим с реальным Firebase:

1. Создайте проект на [Firebase Console](https://console.firebase.google.com/).
2. Включите **Email/Password Authentication** в разделе Authentication.
3. Создайте базу данных **Cloud Firestore** в тестовом режиме.
4. Зарегистрируйте веб-приложение в настройках проекта Firebase и скопируйте конфигурационные ключи.
5. Создайте файл `.env` в корневой директории вашего проекта и заполните его следующими переменными:

```env
VITE_FIREBASE_API_KEY=ваш_api_key
VITE_FIREBASE_AUTH_DOMAIN=ваш_project_id.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=ваш_project_id
VITE_FIREBASE_STORAGE_BUCKET=ваш_project_id.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=ваш_sender_id
VITE_FIREBASE_APP_ID=ваш_app_id
```

6. Перезапустите сервер разработки. Приложение автоматически обнаружит переменные и переключится на работу с базой данных и авторизацией Firebase.

## Сборка для публикации

Чтобы скомпилировать фронтенд для деплоя:

```bash
npm run build
```

Сборка будет сохранена в папку `dist`. Папки `dist` и `netlify` готовы для развертывания на хостинге Netlify.
